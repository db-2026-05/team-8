## 📐 Tasks
For your chosen variant, complete the following:

| # | Task | Details |
|---|------|---------|
| **01** | ⚙️ **Create at least 3 functions** | Each function must encapsulate reusable logic or a calculation relevant to your domain |
| **02** | 📥 **Create SELECT/INSERT procedures** | Write at least 2 stored procedures that perform `SELECT` and/or `INSERT` operations |
| **03** | 🔄 **Create UPDATE procedures** | Write at least 2 stored procedures that perform `UPDATE` operations |
| **04** | 📝 **Document every routine** | Add comments for each function/procedure: purpose, parameters, and expected behavior |
| **05** | 🧪 **Include test calls** | Add example invocations to verify each function and procedure works correctly |
| **06** | 🛡️ **Optional error handling** | Demonstrate `EXCEPTION` handling or transaction management where relevant |
| **07** | 🐘 **Ensure PostgreSQL compatibility** | All routines must execute in PostgreSQL without errors |
| **08** | 📄 **Submit one SQL file** | Submit all functions and procedures as a single `.sql` file |

> [!IMPORTANT]
> Functions and procedures must be meaningfully connected to your project domain — not generic placeholders.

> [!TIP]
> Group the script into sections: functions → SELECT/INSERT procedures → UPDATE procedures → test calls.
