## 📐 Tasks
For your chosen variant, complete the following:

| # | Task | Details |
|---|------|---------|
| **01** | 🧾 **Create insertion scripts** | Write comprehensive `INSERT` scripts for all required tables |
| **02** | 🔢 **Populate enough data** | Each table must contain at least **10 records** with meaningful values |
| **03** | 🔄 **Add updates/deletes if needed** | Include `UPDATE` and `DELETE` scripts when they are relevant to the table logic |
| **04** | 🧠 **Explain skipped operations** | If updates/deletes are not relevant for a table, briefly explain why in documentation |
| **05** | ✅ **Support app behavior and constraints** | DML should help test constraints and cover core functional scenarios |
| **06** | 🧹 **Keep scripts readable** | Add comments by section so scripts are easy to review and execute |
| **07** | 🐘 **Ensure PostgreSQL compatibility** | Scripts must run in PostgreSQL without manual adaptation |
| **08** | 📄 **Submit one SQL file** | Submit DML as a single `.sql` file unless instructed otherwise |

> [!IMPORTANT]
> Use realistic and, where possible, anonymized data that reflects your selected project variant.

> [!TIP]
> Group DML into blocks (reference data, core entities, transactional data) to simplify debugging and reruns.
