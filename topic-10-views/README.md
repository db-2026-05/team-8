## 📐 Tasks
For your chosen variant, complete the following:

| # | Task | Details |
|---|------|---------|
| **01** | 📄 **Create horizontal view** | Select specific columns (example: titles and authors only) |
| **02** | 🎯 **Create vertical view** | Filter rows by condition (example: only overdue records) |
| **03** | ⚖️ **Create mixed view** | Combine selected columns and filtered rows |
| **04** | 🔗 **Create join-based view** | Build at least one view joining multiple tables |
| **05** | 🧠 **Create subquery-based view** | Add at least one view using a subquery |
| **06** | 🔀 **Create UNION-based view** | Combine compatible result sets with `UNION` |
| **07** | 🧩 **Create layered view** | Add one view that selects from another view |
| **08** | ✅ **Use CHECK OPTION view** | Add one updatable view with `WITH CHECK OPTION` |
| **09** | 🧹 **Document each view** | Add comments explaining purpose and relation to the overall design |
| **10** | 🐘 **Ensure PostgreSQL compatibility** | Script must run without errors in PostgreSQL |
| **11** | 📄 **Submit one SQL file** | Submit all views as a single `.sql` file |
| **12** | 🔎 **Optional output demo** | Optionally add sample `SELECT` statements for each view |

> [!IMPORTANT]
> Each view should have a clear purpose and connect directly to your project domain and schema design.

> [!TIP]
> Name views consistently (for example, by function or feature area) and group them by type to simplify review.
